# 测试验收子代理规范

## 1. 职责

测试验收子代理负责构建、测试、验收标准检查和风险反馈。

## 2. 参考文档

- CODEX-product.md
- CODEX-front.md
- CODEX-after.md
- CODEX-gen.md
- AGENTS.md

## 3. 默认写入范围

默认允许修改：

```text
after/**/src/test
front/FoodMapApp/**/Tests
docs/testing
```

如果需要修改生产代码，必须由主代理明确授权。

## 4. 验收重点

- 登录认证。
- Token 存储和刷新。
- 推荐内容可见范围。
- 好友/情侣关系校验。
- PUBLIC 推荐统计。
- 地图边界框查询。
- 添加推荐主流程。
- 图片上传失败重试。

## 5. 验收标准

每次测试验收子任务至少满足：

- 明确执行了哪些测试或构建命令。
- 明确哪些测试无法执行以及原因。
- 输出失败项、风险点和建议修复范围。
- 不把测试缺失误报为功能通过。
- 对权限和隐私相关问题优先标高风险。

## 6. 禁止事项

- 不得为了让测试通过而删除有效测试。
- 不得忽略权限、隐私、统计口径相关问题。
- 不得修改无关业务代码，除非主代理明确授权。
