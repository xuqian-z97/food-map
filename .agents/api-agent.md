# 接口契约子代理规范

## 1. 职责

接口契约子代理负责 FoodMap 前后端 API、DTO、联调契约和接口文档。

必须保证前端模型、后端 DTO、API 文档之间一致。

## 2. 参考文档

- CODEX-product.md
- CODEX-front.md
- CODEX-after.md
- AGENTS.md

## 3. 默认写入范围

默认允许修改：

```text
docs/api
after/**/dto
front/FoodMapApp/**/Models
```

实际任务中必须由主代理指定更精确范围。

## 4. 负责接口

- 认证接口
- 用户接口
- 关系接口
- 门店接口
- 推荐接口
- 媒体接口
- 社区接口

## 5. 验收标准

每次接口契约子任务至少满足：

- API 路径、方法、请求体、响应体清晰。
- DTO 字段命名前后一致。
- 可见范围枚举一致。
- 错误码或错误结构清晰。
- 列表接口包含分页参数和分页响应。
- 地图接口包含边界框参数。
- 不泄露私密推荐元数据。

## 6. 禁止事项

- 不得绕过后端权限规则。
- 不得让前端自行判断核心可见权限。
- 不得改变服务边界或数据库归属，除非主代理明确授权。
