# API 契约检查清单

## 1. 基础结构

- [ ] API 所属服务明确。
- [ ] API 路径符合 `/api/...` 外部路径约定。
- [ ] 内部服务接口使用 `/internal/...` 或清晰内部路径。
- [ ] HTTP 方法正确。
- [ ] 请求参数、请求体、响应体完整。
- [ ] 错误结构清晰。

## 2. 认证和授权

- [ ] 写接口从 Token 获取当前用户身份。
- [ ] 不允许前端传入 userId 覆盖当前登录用户。
- [ ] 后端校验资源归属。
- [ ] 后端校验推荐可见范围。
- [ ] API 不泄露私密推荐元数据。

## 3. 可见范围枚举

必须保持一致：

```text
PRIVATE
SPECIFIC_USERS
FRIENDS
COUPLE
GROUP
PUBLIC
```

检查项：

- [ ] 后端 DTO 使用同一组枚举。
- [ ] 前端模型使用同一组枚举。
- [ ] 文档描述与代码一致。
- [ ] `PUBLIC` 才进入全站统计。

## 4. 分页

列表接口必须包含：

- [ ] pageIndex 或 cursor。
- [ ] pageSize。
- [ ] total 或 hasMore。
- [ ] items。

## 5. 地图查询

地图接口必须支持边界框：

```text
bbox=minLng,minLat,maxLng,maxLat
```

检查项：

- [ ] 支持 zoom 或可推导缩放层级。
- [ ] 支持 scope：MINE、FRIENDS、COUPLE、PUBLIC、GROUP。
- [ ] 支持标签、关键词等筛选。
- [ ] 返回点位摘要不泄露不可见推荐。
